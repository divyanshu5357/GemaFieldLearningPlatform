=====================================
QUERY CHANGES:
=====================================

OLD (Global fetch):
  SELECT * FROM learning_challenges
  WHERE course_id = ? AND is_published = TRUE

NEW (Per-lesson fetch):
  SELECT * FROM learning_challenges
  WHERE course_id = ? 
    AND lesson_id = ? 
    AND is_published = TRUE

This ensures:
- Python course shows ONLY Python challenges
- Each lesson shows ONLY its challenges
- No cross-course contamination

=====================================
HOW IT WORKS NOW:
=====================================

1. Student opens course
2. FirstLesson automatically selected
3. New useEffect fires with selectedLessonId
4. Fetches challenges for that lesson + course
5. UI shows only matching challenges
6. When student clicks different lesson:
   - selectedLessonId updates
   - useEffect fires again
   - New challenges load
   - Previous selection cleared

=====================================
VERIFICATION QUERIES:
=====================================

-- Check challenges are lesson-specific
SELECT 
  c.title as course,
  l.title as lesson,
  COUNT(ch.id) as challenges
FROM courses c
LEFT JOIN lessons l ON l.course_id = c.id
LEFT JOIN learning_challenges ch ON ch.lesson_id = l.id
WHERE ch.is_published = TRUE
GROUP BY c.id, l.id, c.title, l.title
ORDER BY c.title, l.order_index;

-- Verify no NULL lesson_ids
SELECT COUNT(*) as null_lesson_count 
FROM learning_challenges 
WHERE lesson_id IS NULL;

-- Check that Python course has correct challenges
SELECT 
  ch.title,
  ch.challenge_type,
  ch.xp_reward,
  l.title as lesson
FROM learning_challenges ch
JOIN lessons l ON l.id = ch.lesson_id
JOIN courses c ON c.id = ch.course_id
WHERE c.title LIKE '%Python%'
ORDER BY ch.display_order;
